# PoolParty

*A Python package for generating oligonucleotide sequence pools.*


## Installation

Clone the repository and install in editable mode:
```bash
git clone https://github.com/jkinney/poolparty.git
cd poolparty
pip install -e .
```

## Overview

PoolParty facilitates the rapid creation of oligonucleotide pool libraries. The core objects in the package Pools. Pools are sequence generators -- they encode rules for constructing sequences on the fly (i.e., through lazy evaluation). Pools can be manipulated like standard strings, including though addition, integer multiplication, indexing, and slicing. This makes designing a library of sequences very similar to designing individual sequences. 

The package currently supports the following pools. 

- `Pool` - Base class with lazy evaluation. Can be initialized with a list of sequences to select from.
- `KmerPool` - Generates k-mer sequences from a specified alphabet.
- `ShuffledPool` - Generates shuffled versions of an input sequence.
- `RandomMutationPool` - Generates randomly mutated sequences with position-specific mutation rates.
- `KMutationPool` - Generates sequences with exactly k mutations.
- `SubseqPool` - Extracts subsequences with configurable tiling.

Each Pool can generate sequences in one of two modes.
- `"random"` mode: sequences are generated randomly with replacement. 
- `"sequential"` mode: unique sequences are generated in a specific order. 

The `generate_library()` method of Pool objects outputs a list of sequences generated by that Pool. As an argument to this method, users can specify which composite sequences are in which mode. In this way, `generate_library()` is able to generate both random libraries of arbitrary size and combinatorially complete libraries. 

Each Pool has a corresponding computation graph, i.e., the set of objects and operations used in its construction. When generating a sequence, the Pool first signals to each component Pool in its computation graph to generate one new sequence. The Pool then recursively combines these component sequences into a single sequence according to the rules in the computtion graph. 

## Example

Create all single mutations of a DNA sequence, each tagged with 5 different random 10 nt barcode:

```python
from poolparty import KMutationPool, KmerPool

# Define the DNA sequence
seq = "ATCGATCGATCGATCGATCGATCGATCGATCGATCG"

# Create pool for single-mutation variants
promoter = KMutationPool(seq, alphabet='ACGT', k=1)

# Create pool for random 10 nt barcode
barcode = KmerPool(length=10, alphabet='ACGT')

# Concatenate mutation with barcode
library = promoter + barcode

# Generate all single mutants, each with a random barcode
seqs = library.generate_library(
    combinatorially_complete_pools=[promoter],
    num_complete_iterations=5
)
```
