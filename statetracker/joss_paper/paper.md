# Summary

StateTracker is a Python library for decomposing the state of a composite object into the values of its components. Many scientific computing tasks involve creating complex combinatorial structures—for example, a library of DNA sequences generated by combining different mutation types, positions, and barcodes. Given the index of an item in such a structure, one often needs to determine which specific choices were made to produce it. StateTracker solves this problem by letting users build a directed acyclic graph (DAG) of states using operations like Cartesian products and disjoint unions. When the user sets the value of a derived state, StateTracker automatically propagates this value through the DAG, computing the corresponding value of every component state. We developed StateTracker to support DNA library generation, but it addresses a general need applicable to experimental design, parameter sweeps, combinatorial testing, and other domains.

# Statement of Need

Given the index of an item in a complex combinatorial structure, how does one trace it back to the states of its components? This problem arises whenever composite objects are constructed from multiple independent choices—and the answer matters for random access (generating item $n$ without iterating through items $0$ to $n-1$), systematic naming (encoding combinatorial origin in each item's identifier), and reproducibility (tracing experimental results back to specific parameter combinations).

We initially developed StateTracker as part of PoolParty, a Python package for generating complex DNA sequence libraries. We subsequently released StateTracker as a standalone package because it addresses a much more general need; PoolParty now includes StateTracker as a dependency.

Fig. 1 illustrates the motivating application: a 40-sequence DNA library comprising different types of variants of a cis-regulatory element (CRE). Each variant is created using a specific mutational process—point mutations, deletions, or insertions—applied at specific positions, and each variant is paired with one of two barcodes. The problem we faced was this: given a value $n$, how do we compute which mutations to introduce into the sequence and which barcode to attach?

# State of the Field

The state decomposition problem arises across many domains—experimental design matrices, parameter sweeps, combinatorial software testing [@combinatorial_testing], and procedural content generation—yet no general-purpose tool exists for solving it. Python provides several utilities for working with combinatorial structures, but none address the value propagation problem that StateTracker solves. The `itertools` module [@python_itertools] provides `product` for Cartesian products and `chain` for concatenation, but these return iterators or tuples without any mechanism to trace a flat index back to component values. NumPy's `unravel_index` and `ravel_multi_index` functions [@numpy] convert between flat and multi-dimensional indices, but only for rectangular array shapes; they cannot represent structures like disjoint unions or nested combinations of products and unions. Manual index arithmetic using `divmod` works for simple products but does not compose—each additional operation (slicing, shuffling, sampling) requires rewriting the index logic from scratch. This gap persists because the problem appears trivial in simple cases; complexity emerges only when multiple operations are composed, by which point ad hoc solutions are deeply embedded in application code. 

# Software Design

StateTracker addresses this gap by constructing a directed acyclic graph (DAG) of states. Users build the DAG by composing leaf states into derived states; when the user sets the value of a derived state, StateTracker automatically propagates this value to all ancestor states, computing the corresponding value of each. This enables arbitrary composition of products, stacks, slices, shuffles, samples, and splits while maintaining correct value tracking throughout.

The basic unit is a State, which represents an object that can take on a finite number of discrete values numbered from 0 to $n-1$. Users first create "leaf" states specifying the number of values, then compose them using operations to build derived states. The following operations are supported:

| Operation | Description |
|-----------|-------------|
| `product` | Cartesian product; all parent states active |
| `stack` | Disjoint union; one parent state active at a time |
| `slice` | Select subset of values |
| `shuffle` | Randomly permute values |
| `sample` | Sample values with or without replacement |
| `split` | Divide into sub-states by count or proportion |
| `repeat` | Cycle through values multiple times |
| `sync` | Synchronize states in lockstep |
| `interleave` | Alternate between states |

All states must be created within a `Manager` context. The Manager serves as a registry that tracks all states created within a session, assigns unique identifiers, and provides convenience methods for bulk operations such as resetting all states or generating iteration tables. While each state independently tracks its own parent relationships, the Manager provides the global view needed for operations that span the entire collection.

Importantly, the computational graph constructed by StateTracker is a DAG, not necessarily a tree—the same state can appear in multiple branches. This enables powerful patterns, such as using a single state to synchronize choices across different parts of a composite structure. However, shared states also introduce the possibility of conflicts: if state `A` appears in both inputs to a `product`, then setting the product's value would require `A` to have two different values simultaneously. StateTracker automatically detects such conflicts at DAG construction time and raises an informative error, preventing subtle bugs that would otherwise arise from inconsistent value propagation.

## Example

![Figure 2: Value decomposition computed by StateTracker for the library in Fig. 1. Each row shows the value of every state when `seq_state` is set to a given value (0–39). The `stack` operation creates mutually exclusive branches: when `mut_state` or `del_state` is active, the insertion-related states show `None`, and vice versa. The `product` operation keeps all parent states active simultaneously—here, `v_state` cycles through its 2 values for each of the 20 CRE variants.](fig2.png)

The following code constructs the state DAG for the sequence library in Fig. 1. Five leaf states represent the component dimensions: mutation index, deletion position, insertion site, insertion position, and barcode variant. These are composed using `product` and `stack` to build the full 40-value state.

```python
from statetracker import Manager, State, product, stack

# Initialize manager
with Manager() as mgr:
    
    # Define leaf states
    mut_state = State(5, name='mut_state')
    del_state = State(5, name='del_state')
    ins_site_state = State(2, name='ins_site_state')
    ins_position_state = State(5, name='ins_position_state')
    v_state = State(2).named('v_state')
    
    # Build composite states
    ins_state = product([ins_position_state, ins_site_state],
        name='ins_state')
    cre_state = stack([mut_state, del_state, ins_state],
        name='cre_state')
    seq_state = product([v_state, cre_state],
        name='seq_state')
    
# Print DAG
seq_state.print_dag('minimal')

# Print sequences
seq_state.get_iteration_df()
```

The `print_dag()` method displays the state hierarchy. The `cre` state stacks three alternatives (5 mutations + 5 deletions + 10 insertions = 20 values), and `seq` takes the product with the 2-value barcode state, yielding 40 total values.

```
seq_counter (n=40)
└── [Product]
    ├── v_counter (n=2)
    └── cre_counter (n=20)
        └── [Stack]
            ├── mut_counter (n=5)
            ├── del_counter (n=5)
            └── ins_counter (n=10)
                └── [Product]
                    ├── ins_position_counter (n=5)
                    └── ins_site_counter (n=2)
```

# Research Impact Statement

StateTracker is the computational engine that enables PoolParty to generate sequences in complex DNA libraries on demand. A PoolParty library typically contains sequences constructed through different combinations of operations—some sequences might have point mutations, others deletions, others insertions—and each operation has internal parameters controlling specifics like mutation positions or insertion sites. When PoolParty needs to generate the $n$th sequence, it must determine two things: which combination of operations to apply, and what parameter values to use for each. StateTracker answers both questions by decomposing $n$ into the values of individual states. Active states indicate which operations are in effect; their values specify the parameters. This decomposition enables PoolParty to generate any sequence directly from its index, without iterating through preceding sequences.

This capability is essential because DNA library design often requires combining sequences generated through fundamentally different processes into a single library. The value decomposition also supports systematic sequence naming, where each name encodes its combinatorial origin (e.g., `del_2.v0` for the second deletion variant with the first barcode), and enables researchers to trace experimental results back to the specific choices that generated each sequence. 

Beyond DNA library design, StateTracker addresses a general need in any domain involving complex combinatorial structures: experimental design matrices, parameter sweeps in simulations, combinatorial testing of software configurations, and procedural content generation. We anticipate that StateTracker will find applications wherever researchers need to decompose flat indices into meaningful component values.
